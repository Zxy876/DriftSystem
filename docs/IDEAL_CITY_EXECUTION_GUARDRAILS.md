 

一、Copilot 执行清单（不可越界点 · 冻结版）

用途说明（给 Copilot）
这是一个 Execution Guardrail Checklist。
你在后续任何文件创建、TODO 占位、迁移记录中，
必须逐条满足以下约束。

⸻

✅ 世界主权与裁决（最高优先级）
	•	世界裁决只存在于后端逻辑层
	•	插件 / mod 不得参与任何裁决判断
	•	裁决结果只能是：Accept / Partial / Reject
	•	裁决必须产生“可记录的解释数据”
	•	禁止用任务完成、事件匹配替代裁决语义

⸻

✅ DeviceSpec（装置逻辑规格）
	•	DeviceSpec 是一等领域对象
	•	DeviceSpec ≠ 任务 ≠ 剧情节点 ≠ 世界补丁
	•	自然语言 → DeviceSpec → 世界裁决 → 表现
	•	禁止自然语言直出世界补丁 / 命令 / 建造
	•	DeviceSpec 必须可被持久化与回溯

⸻

✅ 算法题与检验机制
	•	算法题只作为“问题结构来源”
	•	禁止任何 OJ / AC / WA / 判分机制
	•	检验 = 世界情境裁决，不是程序正确性
	•	裁决输出是“世界态度”，不是分数

⸻

✅ 表现层 / 执行层（包括 mod）
	•	表现层只消费裁决结果
	•	表现层不可影响裁决结论
	•	mod 只能用于：
	•	现象呈现（题面 / 可视化）
	•	已被接受方案的执行
	•	无任何 mod 情况下系统必须可运行

⸻

✅ 数据与记忆
	•	裁决历史必须可进入世界记忆系统
	•	裁决结果可影响阶段 / 情绪 / 推荐
	•	自由建造不写入裁决历史
	•	裁决失败必须可回溯（诊断，而非报错）

⸻

🚫 明确禁止的工程行为（红线）
	•	❌ 把 DeviceSpec 实现为一次性任务
	•	❌ 用剧情分支假装裁决
	•	❌ 让插件决定“是否接受方案”
	•	❌ 让 mod 先执行再补裁决
	•	❌ 为了方便直接“生成机器”

一旦出现以上任一行为，视为偏离世界愿景。
