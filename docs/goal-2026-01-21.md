	•	✅ 默认 MODEL_CALLS_ENABLED=false
	•	✅ 每次实现前先阅读现有代码与文档，确认无冲突再动手
	•	✅ 仅在 VS Code Workspace /Users/zxydediannao/DRIFT_SCIENCELINE 内操作
	•	✅ 任何脚本执行均在 dry-run 说明下进行（当前环境不可实跑 Minecraft / mineflayer）
	•	⚠️ 若遇到阻塞依赖（如 sqlite 文件、真实服务器），以 stub / 注释说明形式落地并在 Issue 中标注 TODO（需明确不阻塞测试）

单兵执行策略
	1. 创建 `notes/v1.21-progress.md`（若不存在）记录每个 Issue 的状态、待办、阻塞原因。
	2. Issue 执行流程：
		•	阅读当前 Issue 需求 → 评估依赖 → 在 notes 中登记。
		•	若需要新目录 / 文件，先确认上游 Issue 是否已创建；若未创建，则在当前 Issue 先落地必要骨架，并注明归属。
		•	实现代码 → 编写或更新对应测试 → 运行相关 pytest / 脚本（能跑的部分） → 记录结果。
		•	整理提交说明（当前流程不执行 git commit，由使用者手动处理）。
	3. 每日结束前更新 `notes/v1.21-progress.md`，列出已完成 Issue、剩余风险、待补证据。
	4. 若需人工验证（Mineflayer / demo），在 docs/freeze_evidence 下添加模板并标记“待外部执行”。
